不懂设计的前端工程师怎么做 OG Image，我来分享一下我的工作流。

这条教程已经发布到我的信息差网站：

先看成果：https://PHCopilot.AI/，推文下面出现的网站封面就是 OG Image。作为 0 设计经验的前端工程师，我觉得这个 OG Image 还过得去。

现在介绍整个流程：

第一步：自己思考

我的思路很简单，就两个要点：
1、这是一个 Chrome 插件产品，OG Image 上应该放一张产品截图；
2、听说 OG Image 上面放一个按钮会增加转化，那我也加一个。

第二步：找 Claude 帮我思考设计

我的 Prompt 结构也很简单：

产品标题：PH Copilot
产品介绍：It's an AI-Powered Chrome Extension for Product Hunt. Quickly overview products and craft standout comments using AI. Ideal for hunters and makers.
现在我要给这个产品设计一个 OG Image，要求图片上有一张使用截图，让用户从图片就能看出来插件用法；还要有一个按钮，因为按钮可以提高点击率。
请你为我思考一下怎么设计比较好。

就这样，Claude 给出了很详细的设计思路，包含尺寸、布局、内容区和信息区分别展示什么、视觉元素、配色等等信息。AI 回复太长，这里就不挂了，你可以到我的文章里看：xxxxxxxxxx这里是url

第三步：到 Figma 找素材

不得不说，Figma 是个设计大宝藏，我搜关键词“Product Hunt”，看了一些设计方法，决定用渐变色背景。本来还想直接用 Figma 来做图，但是确实手残，用不来。

第四步：用创客贴/稿定设计做图

创客贴/稿定设计都是提供了很多模板和设计素材的网站，功能差不多，就不说是哪个平台了，因为我有会员，所以不用考虑素材商用的问题。我在平台里找了一圈模板，没有找到满意的，所以自己创建空白模板（按 AI 提供的尺寸），再把 Figma 上的开源渐变色背景下载了放这边用。剩下的步骤就是根据 AI 提供的设计方案一步步做了。

第五步：压缩图片

设计网站导出的图片都很大，用来做 OG Image 可不合适，压缩图片是必须的。我现在找到的最大压缩方法是：

- 先到 https://tinypng.com/ 压缩一遍
- 再到 https://tiny-img.com/webp/ 压缩+转成 webp 格式

两个压缩步骤反过来也行。

第六步：体验一下

有两种方法：

- 使用 AITDK 插件（Chrome 插件商城搜 AITDK 就行，xxxxxx网站里提供链接xxxxxx）的 Social 功能可以看到 OG Image
- 把网站地址粘贴到推特输入框，就会显示带 OG Image 的链接卡片

第起步：不断修改

我微调了 4 个版本，最终确定了现在你们看到的这张图。

总结

设计也是个孰能生巧的手艺活，这次是第一次自己做 OG Image，耗时 3 小时，以后多做几次应该可以控制在 30 分钟内搞定。
